\section{Experimental Setup}
\label{sec:setup}

\paragraph{Base model.}
All experiments use Qwen2.5-7B~\citep{qwen2024} with bfloat16 weights, eager attention, and no quantization. The model has 28 transformer layers, 28 query heads, 4 key-value heads (grouped-query attention), and head dimension 128. We adapt the top-third of layers (layers 19--27) for both KV memory injection and W$_{\mathrm{cons}}$ LoRA.

\paragraph{Dataset.}
We construct a controlled benchmark of 200 synthetic facts spanning 10 templates: corporate financials, scientific discoveries, city foundings, regulatory protocols, world records, technology architectures, medical trials, sports records, album releases, and geological events. Each fact is a single sentence containing fictional entity names, specific numerical values, and contextual details. Templates are deliberately structured so that the surprising tokens (entity names like ``Nimbus Holdings'') cluster at the start of each fact, with factual content (revenue figures, dates, percentages) following. Each fact is paired with a question prompt and a list of three keywords expected in a correct answer. The dataset is generated by deterministic seeded Python code; see Appendix for the generator.

We deliberately use synthetic facts rather than natural documents to isolate single-exposure encoding from prior-knowledge confounds. Real-world facts may have been seen during pretraining, making it impossible to distinguish ``the model recalls because it learned'' from ``the model recalls because it always knew.''

\paragraph{Evaluation metrics.}
We measure three forms of recall and one preservation metric:

\textbf{Multiple-choice accuracy (MC):} for each fact, the correct keyword and three same-template distractors are presented as options A--D in a randomized order seeded by fact ID. We compute the model's logit at each option's first token following ``Answer:'' and report the fraction where the argmax matches the correct option. We also log $P(\mathrm{correct})$ and $P(\mathrm{correct} \mid \mathrm{wrong})$ via softmax over the four option-letter logits, providing a calibration view sensitive to partial encoding.

\textbf{Cloze accuracy:} the prefix of the fact text up to (but not including) the first keyword is provided as a prompt; the model generates 20 tokens; we score 1 if the keyword appears in the generation, 0 otherwise.

\textbf{Free-form delayed recall accuracy (DRA):} the fact's question prompt (e.g., ``What was Nimbus Holdings' Q3 revenue?'') is provided; the model generates 50 tokens with temperature 0.7; we compute the fraction of expected keywords appearing in the generation. The aggregate DRA is the mean across facts.

\textbf{Base capability preservation (BCP):} we measure perplexity on three held-out control texts (capital of France, water boiling temperature, Python's history) before and after each modification, and report the ratio $\mathrm{BCP} = \mathrm{PPL}_{\mathrm{after}} / \mathrm{PPL}_{\mathrm{before}}$. $\mathrm{BCP} \leq 1.05$ is the preservation threshold (no meaningful degradation); $\mathrm{BCP} > 2$ indicates substantial damage.

\paragraph{Baselines.}
\textbf{Base model} is the unmodified Qwen2.5-7B. \textbf{Naive LoRA} is a single LoRA adapter (rank 16, $\alpha=32$, applied to V/O projections of the same top-third layers as W$_{\mathrm{cons}}$) trained directly on the 200 facts via uniform-with-replacement sampling for 100 gradient steps with AdamW (lr $10^{-4}$, weight decay 0.01, batch size 32). This matches SLEEP's compute budget but applies no safety constraints.

\paragraph{Hyperparameters.}
All non-overridden values track the formalization defaults. We document four hyperparameter overrides applied during the empirical work, each with its empirical justification logged at the time the change was made (see logbook entries cited in Appendix~A): $\alpha_{\mathrm{slow}} = 10^{-4}$ (precision floor under bfloat16; the $10^{-5}$ default produces zero detectable parameter movement), $\delta_{\max} = 0.01$ (model-scale-dependent; $10^{-3}$ saturates within 30 steps on 7B), KV top-$k = 64$ (must scale with episode size; episodes are $\sim 30$--40 tokens so $k=16$ retrieves only fragments), and quality-check $\mu_{\mathrm{surprise}}$ disabled (the formalization's absolute-surprise threshold suffers from a generator-discriminator gap on self-generated replays). Full hyperparameter table in Appendix~A.

\paragraph{Compute.}
All experiments run on a single NVIDIA RTX 5090 (32GB VRAM) via RunPod. Total compute for the experiments reported in this paper: approximately 4 GPU-hours, totaling roughly USD \$4 in cloud cost. Each end-to-end run (200 facts, full sleep cycle, complete evaluation suite) takes 8--15 minutes. We log every run's command line, configuration, output, and resulting metrics; these are released alongside the implementation.

\paragraph{Reproducibility.}
The implementation comprises approximately 3,500 lines of Python across six modules (tagging, prp, weights, sleep\_engine, orchestrator, evaluation), with 297 unit and integration tests. The 36-question formalization and the empirical amendments derived from it are released as a separate document to enable independent reproduction. The exact RunPod instance image, Python version, and \texttt{transformers} library version (5.7.0) are documented; \texttt{attn\_implementation=eager} is forced because the SDPA attention path handles 4D additive masks differently and silently degrades the gating mechanism.
